README.md :
## AURI V1 — Smart Media Downloader


> **Version 2.0** | Python 3.11+ | yt-dlp powered | Rich terminal UI


---


### Features


- ✅ 10 quality presets (4K → 144p)
- ✅ Paste a single URL, multiple comma-separated URLs, or a full playlist URL
- ✅ Per-video or bulk quality selection
- ✅ 8 parallel download workers for maximum speed
- ✅ Real-time Rich progress bars (speed, %, ETA, size)
- ✅ Pause **P** / Resume **R** / Stop **S** keyboard controls
- ✅ SQLite database logging (every download, session, and error)
- ✅ Beautiful post-session report table
- ✅ Loop: download more or exit gracefully


---


### Setup


```bash
# 1. Create & activate a virtual environment (recommended)
python -m venv .venv
.venv\\\\Scripts\\\\activate      # Windows
# source .venv/bin/activate  # macOS/Linux


# 2. Install dependencies
pip install -r requirements.txt


# 3. (Windows only) Install ffmpeg — required for video+audio merging
# Download from https://ffmpeg.org/download.html and add to PATH
```


---


### Usage


```bash
python main.py
```


**Flow:**
1. Paste URL(s) — single, comma-separated, or playlist
2. Choose quality scope: **(a)** all videos or **(i)** per-video
3. Pick quality from the numbered menu (1–10)
4. Watch downloads run in parallel with live progress bars
5. Press **P** to pause, **R** to resume, **S** to stop
6. View the session report table automatically when done
7. Choose to download more or exit


---


### Project Structure


```
AURI V1/
├── main.py          ← Entry point & session loop
├── downloader.py    ← yt-dlp wrapper with Rich progress hooks
├── database.py      ← SQLite3 models + CRUD (thread-safe)
├── ui.py            ← All Rich UI components
├── config.py        ← App settings and quality map
├── requirements.txt ← All dependencies
├── auri.db          ← SQLite database (auto-created on first run)
└── downloads/       ← Default save location
```


---


### Requirements


See `requirements.txt`. Key packages:


| Package   | Purpose                          |
|-----------|----------------------------------|
| yt-dlp    | Media downloading engine         |
| rich      | Terminal UI, progress bars, tables|
| keyboard  | p/r/s hotkey detection           |


---


### Notes


- `auri.db` is created automatically on first run in the project folder.
- If `keyboard` is not installed, hotkeys are disabled (downloads still work).
- For best video quality, ensure **ffmpeg** is installed and on your PATH.

|---|---|

@test("GUI-16", "CSV Export to ZIP")
def test_csv_export():
    from core.database import AuryDB
    import zipfile, tempfile, os
    db = AuryDB(str(TEST_DB))
    # Manually test the export logic
    tables = ["downloads","sessions","platforms","settings","tags"]
    zip_path = Path(tempfile.mkdtemp()) / "test_export.zip"
    with zipfile.ZipFile(zip_path, "w") as zf:
        for table in tables:
            rows = db.conn.execute(
                f"SELECT * FROM {table}").fetchall()
            cols = [d[0] for d in db.conn.execute(
                f"SELECT * FROM {table}").description or []]
            csv_content = ",".join(cols or []) + "\n"
            for row in rows:
                csv_content += ",".join(str(c) for c in row)+"\n"
            zf.writestr(f"{table}.csv", csv_content)
    assert zip_path.exists(), "ZIP file not created"
    with zipfile.ZipFile(zip_path) as zf:
        assert len(zf.namelist()) == 5, \
            f"Expected 5 CSVs, got {len(zf.namelist())}"
    db.conn.close()
    return f"ZIP export OK: {len(tables)} tables → CSVs"
